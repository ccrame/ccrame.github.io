The this keyword is very common in JavaScript and understanding how it works is important in order to write better code. If you are new to JavaScript, there are a few basic patterns you can memorize to gain a quick understanding of the keyword this.

In the example below, this refers to the object to the left of the dot. To be more precise, this refers to “myObj”:


  var myObj = {};
  myObj.test = function(){
    return this;
  };
  
  myObj.test(); //myObj

However, in certain cases, the above does not work. An example of this can be seen when using setTimeout. Passing “myObj.test” into setTimeout is equivalent to passing the specific set of instructions included within that function. “myObj” is not passed at all into the setTimeout function. When setTimeout is finally invoked, it is called from the window object (if using a browser), and thus, returns the window object.


  setTimeout(myObj.test,1000); //returns the window object

To avoid using this incorrectly, one must know the location of this during CALL TIME (where the function is invoked, not where it is referenced).
